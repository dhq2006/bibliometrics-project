# 检索策略说明（生成式AI在人文社科与人文计算领域的应用）

## 一、研究问题
定量分析2015-2025年生成式AI（AIGC）在人文社科与人文计算领域的应用研究热点、发文趋势、核心技术方向与研究团队，为后续文献计量分析、前沿趋势追踪奠定数据基础。

## 二、数据源选择依据
选择Web of Science Core Collection作为唯一数据源：
1. 权威性：全球顶级英文学术数据库，覆盖人文社科、数字人文等多领域，收录的AIGC相关文献质量高、被引数据完整；
2. 字段完整性：包含作者、机构、参考文献、DOI、关键词等核心字段，满足文献计量分析（关键词共现、合作网络、共被引分析、突现检测）的所有需求；
3. 检索灵活性：支持高级布尔检索与TS主题检索，可精准匹配AIGC技术+人文社科场景的核心概念，避免无关文献；
4. 格式通用性：导出的纯文本格式（savedrecs.txt）可直接用于Python、CiteSpace、VOSviewer等计量分析工具，无需额外格式转换。

## 三、检索式设计规范
### 1. 检索字段：TS（Topic Search，主题检索）
选择TS字段（覆盖标题、摘要、关键词），核心理由：平衡查全率与查准率——
- 避免仅选标题（TI）或关键词（DE）导致的文献遗漏；
- 避免选全字段（ALL）导致的检索范围失控，保证文献与研究主题高度相关。

### 2. 概念拆分：Object（研究对象） + Context（应用场景）
将研究问题拆分为「二元核心概念」，用AND连接：
- Object（研究对象）：生成式AI（AIGC）核心技术，覆盖LLM/GPT/扩散模型/GAN等主流技术；
- Context（应用场景）：人文社科与人文计算相关场景，覆盖数字人文、人文计算、各学科研究、文本挖掘、语料库分析等。

### 3. 筛选条件：多层限定，保证数据一致性与有效性
- 时间窗：2015-2025（完整覆盖AIGC领域发展周期）；
- 文献类型：Article、Review（仅保留学术文献，保证数据学术性）；
- 语言：English（避免跨语言数据处理偏差）。

### 4. 布尔逻辑：严格遵循WOS检索规则
- 同义词组内用OR：扩展检索范围，覆盖同一概念的不同表述；
- 概念组间用AND：收紧检索范围，确保文献同时包含AIGC技术与人文社科场景。

## 四、同义词扩展依据
1. 领域经典文献：参考AIGC高被引综述，收录领域通用术语；
2. 行业通用表述：包含官方缩写（AIGC、LLM、GAN）与全称，适配新兴领域用语习惯；
3. 口径一致性：同义词表与检索式一一对应，保证分析一致性。

## 五、检索式版本记录
v0.1（2026-03-28）：初始检索式，完成「技术+场景」二元概念拆分，扩展25+同义词，限定时间、文献类型、语言，适配WOS高级检索语法，无逻辑错误。

## 六、数据质量保障措施
1. 检索式可复现：检索式完整保存至config/query.yaml，可直接复现文献集合；
2. 数据字段完整：导出全字段数据，满足后续文献计量分析需求。
